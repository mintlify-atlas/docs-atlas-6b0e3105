---
title: Get Quotation
api: GET /api/cotizaciones/:id
---

Retrieves a single quotation by ID with related data (client, origin, via, seller).

## Authorization

Requires active session. Non-admin users can only access quotations they created.

## Path Parameters

<ParamField path="id" type="number" required>
  Quotation ID
</ParamField>

## Response

<ResponseField name="data" type="object">
  Quotation object with joined data

    </ResponseField>

    </ResponseField>
    
    <ResponseField name="detalle" type="string">
      Cargo description
    </ResponseField>
    
    <ResponseField name="imo" type="string">

    <ResponseField name="semana" type="number">

    <ResponseField name="mes" type="number">

    <ResponseField name="anio" type="number">

      Seller/user ID
    </ResponseField>
    
    <ResponseField name="vendedor_nombre" type="string">

    </ResponseField>
  </Expandable>
</ResponseField>

## Error Responses

| Status | Condition |
|--------|----------|
| 401 | No active session |
| 403 | Non-admin user attempting to access another user's quotation |
| 404 | Quotation ID does not exist |

## Notes

- **Role-based access:** Admin users can access any quotation. Non-admin users can only access quotations where `vendedor_id` matches their user ID.
- The response includes joined data from related tables (clients, locations, users)
- Field names in the response use Spanish naming convention for frontend compatibility
