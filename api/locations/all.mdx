---
title: 'All Locations (Unified)'
api: 'GET/POST/PUT/DELETE /api/locations'
---

Manage all locations (both origins and vias) through a unified endpoint. This endpoint operates on the `locations` table without filtering by type.

<Note>
  For type-specific operations, use the dedicated `/api/origenes` or `/api/vias` endpoints which filter by `location_type`.
</Note>

## GET - List All Locations

Retrieves all locations regardless of type (both "origen" and "via").

### Response

<ResponseField name="id" type="integer">
  Unique identifier for the location
</ResponseField>

<ResponseField name="name" type="string">
  Location name
</ResponseField>

<ResponseField name="country" type="string">
  Country name
</ResponseField>

<ResponseField name="region" type="string">
  Region or state (nullable)
</ResponseField>

<ResponseField name="location_type" type="string">
  Type of location: "origen" or "via"
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether the location is active
</ResponseField>

### Request Example

```bash cURL
curl -X GET https://api.iclcotizaciones.com/api/locations \
  -H "Cookie: session=your_session_token"
```

### Response Example

```json 200
[
  {
    "id": 1,
    "name": "Shanghai",
    "country": "China",
    "region": "Shanghai",
    "location_type": "origen",
    "is_active": true
  },
  {
    "id": 10,
    "name": "Buenos Aires",
    "country": "Argentina",
    "region": "Buenos Aires",
    "location_type": "via",
    "is_active": true
  }
]
```

```json 401
{
  "error": "No autorizado"
}
```

## POST - Create Location

Creates a new location of any type. This endpoint requires admin permissions.

<Warning>
  **Admin Only**: Only users with admin roles (DIRECTOR, GERENTE, ADMINISTRACION) can create locations.
</Warning>

### Request Body

<ParamField body="name" type="string" required>
  Location name
</ParamField>

<ParamField body="country" type="string" required>
  Country name
</ParamField>

<ParamField body="region" type="string" optional>
  Region or state
</ParamField>

<ParamField body="location_type" type="string" required>
  Type of location. Must be either "origen" or "via"
</ParamField>

<ParamField body="is_active" type="boolean" optional default={true}>
  Whether the location is active
</ParamField>

### Request Example

```bash cURL
curl -X POST https://api.iclcotizaciones.com/api/locations \
  -H "Cookie: session=your_session_token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Hamburg",
    "country": "Germany",
    "region": "Hamburg",
    "location_type": "via",
    "is_active": true
  }'
```

### Response Example

```json 201
{
  "id": 30,
  "name": "Hamburg",
  "country": "Germany",
  "region": "Hamburg",
  "location_type": "via",
  "is_active": true
}
```

```json 403
{
  "error": "No autorizado"
}
```

## PUT - Update Location

Updates an existing location by ID. This endpoint requires admin permissions.

<Warning>
  **Admin Only**: Only users with admin roles (DIRECTOR, GERENTE, ADMINISTRACION) can update locations.
</Warning>

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the location to update
</ParamField>

### Request Body

<ParamField body="name" type="string" required>
  Location name
</ParamField>

<ParamField body="country" type="string" required>
  Country name
</ParamField>

<ParamField body="region" type="string" optional>
  Region or state
</ParamField>

<ParamField body="location_type" type="string" required>
  Type of location. Must be either "origen" or "via"
</ParamField>

<ParamField body="is_active" type="boolean" optional>
  Whether the location is active
</ParamField>

### Request Example

```bash cURL
curl -X PUT https://api.iclcotizaciones.com/api/locations/30 \
  -H "Cookie: session=your_session_token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Hamburg Port",
    "country": "Germany",
    "region": "Hamburg",
    "location_type": "via",
    "is_active": true
  }'
```

### Response Example

```json 200
{
  "id": 30,
  "name": "Hamburg Port",
  "country": "Germany",
  "region": "Hamburg",
  "location_type": "via",
  "is_active": true
}
```

```json 403
{
  "error": "No autorizado"
}
```

## DELETE - Delete Location

Deletes a location by ID. This endpoint requires admin permissions.

<Warning>
  **Admin Only**: Only users with admin roles (DIRECTOR, GERENTE, ADMINISTRACION) can delete locations. This is a hard delete and cannot be undone.
</Warning>

### Path Parameters

<ParamField path="id" type="integer" required>
  The unique identifier of the location to delete
</ParamField>

### Request Example

```bash cURL
curl -X DELETE https://api.iclcotizaciones.com/api/locations/30 \
  -H "Cookie: session=your_session_token"
```

### Response Example

```json 200
{
  "ok": true
}
```

```json 403
{
  "error": "No autorizado"
}
```
