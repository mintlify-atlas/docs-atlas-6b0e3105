---
title: 'Via Locations'
api: 'GET/POST /api/vias'
---

Manage via (transit) locations for shipments.

## GET - List Via Locations

Retrieves all via locations.

### Response

<ResponseField name="id" type="integer">
  Unique identifier for the location
</ResponseField>

<ResponseField name="name" type="string">
  Location name
</ResponseField>

<ResponseField name="country" type="string">
  Country name
</ResponseField>

<ResponseField name="region" type="string">
  Region or state
</ResponseField>

<ResponseField name="location_type" type="string">
  Always "via" for this endpoint
</ResponseField>

<ResponseField name="is_active" type="boolean">
  Whether the location is active
</ResponseField>

### Request Example

```bash cURL
curl -X GET https://api.iclcotizaciones.com/api/vias \
  -H "Cookie: session=your_session_token"
```

### Response Example

```json 200
[
  {
    "id": 10,
    "name": "Buenos Aires",
    "country": "Argentina",
    "region": "Buenos Aires",
    "location_type": "via",
    "is_active": true
  },
  {
    "id": 11,
    "name": "Montevideo",
    "country": "Uruguay",
    "region": "Montevideo",
    "location_type": "via",
    "is_active": true
  }
]
```

```json 401
{
  "error": "No autorizado"
}
```

## POST - Create Via Location

Creates a new via location. This endpoint requires admin permissions.

<Note>
  **Admin Only**: Only users with admin roles (DIRECTOR, GERENTE) can create via locations.
</Note>

### Request Body

<ParamField body="name" type="string" required>
  Location name
</ParamField>

<ParamField body="country" type="string" required>
  Country name
</ParamField>

<ParamField body="region" type="string" optional>
  Region or state
</ParamField>

<ParamField body="is_active" type="boolean" optional default={true}>
  Whether the location is active
</ParamField>

### Request Example

```bash cURL
curl -X POST https://api.iclcotizaciones.com/api/vias \
  -H "Cookie: session=your_session_token" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Santos",
    "country": "Brazil",
    "region": "São Paulo",
    "is_active": true
  }'
```

### Response Example

```json 201
{
  "id": 25,
  "name": "Santos",
  "country": "Brazil",
  "region": "São Paulo",
  "location_type": "via",
  "is_active": true
}
```

```json 403
{
  "error": "No autorizado"
}
```
